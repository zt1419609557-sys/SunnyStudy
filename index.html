原理图

一次请求网站，背后到底发生了什么

把浏览器、DNS、服务器、数据库和部署之间的关系，压缩成一张好懂的流程图。

浏览器输入网址，开始一次访问
→
DNS / 域名把门牌号翻译成 IP 地址
→
服务器 / API负责处理请求、返回内容
→
数据库保存文章、用户、订单等数据

浏览器像前台

负责展示页面、收集输入、发起请求。

后端像厨房

负责规则、权限、计算和数据读写。

数据库像仓库

把信息分类存好，方便查询、修改、备份。

域名像门牌号

让别人不用记复杂 IP，也能找到你的网站。
